### [섹션 1: 과거 vs 현재 - 무엇이 달라졌나] (1:00 ~ 2:30)

**화면:**
- 화면 분할: 왼쪽 "과거", 오른쪽 "현재"
- 과거: 책, 노트, 폴더 가득한 예제 파일
- 현재: AI 채팅창, 간단한 메모

**나레이션:**
```
먼저, 과거와 현재를 비교해볼게요.

[과거의 공부법]

1. 문법을 암기합니다
   - for문, if문, 함수 선언... 전부 외워야 했어요

2. 예제 코드를 보관합니다
   - "나중에 쓸 일 있을 거야" 하면서 폴더에 저장
   - 정작 필요할 때는 어디 있는지 못 찾죠

3. 에러가 나면 책을 뒤집니다
   - 또는 구글에서 비슷한 문제 찾기
   - 시간이 오래 걸립니다

[현재의 공부법 - AI와 함께]

1. 문법의 "존재"만 알면 됩니다
   - for문이 뭔지, 언제 쓰는지만 이해
   - 정확한 형식은 AI에게 물어보면 돼요

2. 예제 코드 보관? 필요 없습니다
   - 필요할 때 AI에게 "~하는 코드 만들어줘"
   - 매번 최신 방식으로 받을 수 있어요

3. 에러가 나면 AI에게 보여줍니다
   - "이 에러 왜 나?" 하면 바로 설명해줘요
   - 해결 방법까지 알려줍니다

핵심은 이겁니다.
"암기"에서 "이해"로, "보관"에서 "질문"으로 바뀐 거예요.
```

```
AI 시대의 코딩 공부법, 3가지 원칙이 있습니다.

[원칙 1: 문법의 "뼈대"만 익힌다]

모든 걸 외울 필요 없어요.
"Python에서 반복하려면 for문을 쓴다"
"조건 분기는 if문이다"
"함수를 만들면 재사용할 수 있다"

이 정도만 알면 됩니다.
정확한 문법? AI가 알려줘요.

[원칙 2: 주요 도구가 "있다"는 것만 안다]

Python으로 웹 크롤링을 하고 싶다면?
"requests나 BeautifulSoup 라이브러리가 있다"
이것만 알면 돼요.

사용법은 AI에게 물어보면 됩니다.
"requests로 웹페이지 가져오는 코드 보여줘"
바로 예제가 나옵니다.

[원칙 3: 작은 것부터 만들면서 배운다]

책을 다 읽고 시작하지 마세요.
간단한 프로젝트를 정하고, 바로 만들기 시작하세요.

모르는 게 나오면? AI에게 물어봅니다.
그 과정에서 자연스럽게 배우게 돼요.

이게 가장 빠른 길입니다.
```

### [섹션 3: 실전 예시 1 - 파일 다루기] (4:00 ~ 5:30)

**화면:**
- VS Code 화면
- AI 채팅창과 코드 편집기 나란히

**나레이션:**
```
이제 실제 예시를 보여드릴게요.

[예시 1: 파일 읽고 쓰기]

예전 방식:
"open 함수에서 'r'은 읽기, 'w'는 쓰기..."
"with 문을 쓰면 자동으로 파일이 닫히고..."
"인코딩은 utf-8로 해야 하고..."
이걸 다 외웠어야 했습니다.

AI와 함께하는 방식:
일단 이것만 알면 됩니다.
"Python에서 파일을 읽고 쓸 수 있다"

그리고 AI에게 물어봅니다.
"data.txt 파일을 읽어서 내용을 출력하는 코드 만들어줘"

[AI가 코드를 생성하는 화면]

with open('data.txt', 'r', encoding='utf-8') as file:
    content = file.read()
    print(content)

이 코드를 받아서 실행해봅니다.
작동하면 성공!

그런데 여기서 멈추면 안 됩니다.
코드를 보면서 이해합니다.

"아, with문을 쓰는구나"
"'r'이 읽기 모드구나"
"encoding을 지정하는구나"

이렇게 실제 코드를 보면서 익히는 게
책으로 외우는 것보다 훨씬 빨리 기억에 남습니다.

다음에 비슷한 작업을 하면?
"아, 파일 읽기... with open 쓰는 거였지"
이 정도만 기억나도 AI에게 정확한 코드를 받을 수 있어요.
```

### [섹션 4: 실전 예시 2 - API 사용하기] (5:30 ~ 7:00)

**화면:**
- 날씨 API 응답 화면
- 코드와 실행 결과

**나레이션:**
```
[예시 2: 외부 API로 날씨 정보 가져오기]

이건 초보자에게 꽤 어려운 주제입니다.
예전에는 이렇게 배웠어요.

1. HTTP 요청이 뭔지 공부
2. requests 라이브러리 설치법
3. JSON 파싱 방법
4. API 키 사용법
... 배울 게 너무 많죠?

AI와 함께하면 이렇게 합니다.

먼저, 이것만 압니다.
"외부 서비스의 데이터를 가져오려면 API를 사용한다"
"Python에서는 requests 라이브러리를 쓴다"

그리고 AI에게 물어봅니다.
"OpenWeatherMap API로 서울 날씨 가져오는 Python 코드 만들어줘"

[AI가 완성된 코드를 보여주는 화면]

import requests

api_key = "your_api_key"
city = "Seoul"
url = f"https://api.openweathermap.org/data/2.5/weather?q={city}&appid={api_key}"

response = requests.get(url)
data = response.json()

print(f"온도: {data['main']['temp']}K")
print(f"날씨: {data['weather'][0]['description']}")

코드를 받고 실행합니다.
API 키만 넣으면 바로 작동해요!

이제 코드를 보면서 배웁니다.
"requests.get으로 데이터를 가져오는구나"
"response.json()으로 JSON을 파싱하는구나"
"data['main']['temp']로 원하는 값을 꺼내는구나"

한 번 해보면, 다른 API도 비슷하게 할 수 있습니다.
유튜브 API, 주식 API, 번역 API...
기본 패턴은 똑같거든요.

이게 AI와 함께 배우는 힘입니다.
어려운 주제도 빠르게 실전에 적용할 수 있어요.
```

### [섹션 5: 실전 예시 3 - 에러 해결하기] (7:00 ~ 8:30)

**화면:**
- 빨간 에러 메시지가 뜬 화면
- AI에게 에러를 보여주고 해결하는 과정

**나레이션:**
```
[예시 3: 에러가 났을 때]

코딩하다 보면 에러는 반드시 납니다.
예전에는 이게 정말 힘들었어요.

에러 메시지를 복사해서 구글에 검색하고,
Stack Overflow에서 비슷한 문제를 찾고,
영어로 된 답변을 해석하고...

AI와 함께라면 이렇게 합니다.

에러가 났습니다.
"TypeError: 'str' object cannot be interpreted as an integer"

AI에게 그대로 보여줍니다.
"이 에러 왜 나? 내 코드는 이거야: for i in range('5'):"

[AI의 답변 화면]

AI가 바로 알려줍니다.
"range() 함수는 정수가 필요한데, 문자열 '5'를 넣었어요.
'5'를 5로 바꾸세요."

for i in range(5):  # 문자열 '5'가 아니라 숫자 5

이렇게 고치면 해결!

그런데 여기서 한 걸음 더 나갑니다.
"왜 이런 에러가 나는지 더 설명해줘"

AI가 설명해줍니다.
"Python에서 '5'는 문자열이고, 5는 정수입니다.
따옴표가 있으면 문자열, 없으면 숫자예요."

이렇게 에러를 통해 배우는 겁니다.
실수할 때마다 지식이 쌓여요.

책에서 "문자열과 정수의 차이"를 읽는 것보다
직접 에러를 만나고 해결하는 게 훨씬 기억에 남습니다.
```

```
[장점 4: 실전 경험이 빨리 쌓입니다]
이론만 아는 게 아니라
실제로 작동하는 코드를 만들면서 배웁니다.
포트폴리오도 빨리 만들 수 있어요.

핵심은요,
"다 알고 시작하기"가 아니라
"만들면서 알아가기"입니다.
```
