# UTTEC Edu Python 교안 작성 가이드라인

## 1. 기본 철학: AI 시대의 학습법

### 핵심 원칙

| 기존 방식 | UTTEC 방식 |
|----------|-----------|
| 문법 암기 → 예제 적용 | **실습 먼저 → 이해 → 확장** |
| 모든 내용을 외워야 함 | **뼈대만 이해하면 됨** |
| 예제 코드 보관 | **필요할 때 AI에게 요청** |
| 책 완독 후 실전 | **바로 만들면서 배움** |

### 학습 흐름

```
[AI에게 코드 요청] → [실행해보기] → [코드 이해하기] → [수정해보기] → [개념 정리]
```

---

## 2. 교안 구조 템플릿

### Day N: [주제]

각 Day는 2~3개의 목표로 구성됩니다.

#### 목표 구성 요소

```markdown
### 목표 N: [목표 제목]

#### 1. 오늘 배울 것 (뼈대)
- 한 문장으로 핵심 개념 설명
- "이런 게 있다"는 것만 알면 됨
- 예: "Python에서 조건에 따라 다른 코드를 실행하려면 if문을 사용합니다"

#### 2. AI에게 요청하기 (프롬프트)
- 학습자가 AI에게 할 질문
- 구체적이고 실용적인 예제 요청
- 예: "숫자를 입력받아서 짝수인지 홀수인지 판단하는 Python 코드를 만들어줘"

#### 3. 코드 이해하기 (핵심 포인트)
- AI가 생성한 코드에서 봐야 할 부분 3~5개
- 각 포인트는 "아, ~구나!" 형태로 깨달음 유도
- 예:
  - "if 다음에 조건을 쓰는구나"
  - "조건 끝에 콜론(:)을 붙이는구나"
  - "들여쓰기로 실행할 코드를 구분하는구나"

#### 4. 수정해보기 (실습)
- 간단한 수정 과제 1~2개
- "3을 5로 바꿔보세요" 같은 구체적 지시
- 직접 코드를 수정하며 문법 체득

#### 5. 정리 (기억할 것)
- 핵심만 2~3줄로 정리
- 다음에 AI에게 질문할 때 쓸 키워드
- 예: "조건 분기는 if문, 콜론(:)과 들여쓰기 필수"
```

## 3. 프롬프트 작성 원칙

### 좋은 프롬프트의 조건

1. **구체적인 목표가 있다**
   - ❌ "if문 예제 보여줘"
   - ✅ "점수를 입력받아서 60점 이상이면 '합격', 미만이면 '불합격' 출력하는 코드 만들어줘"

2. **실생활과 연결된다**
   - ❌ "for문으로 1부터 10까지 출력해줘"
   - ✅ "쇼핑 목록에 있는 물건들을 하나씩 출력하는 코드 만들어줘"

3. **점진적으로 확장 가능하다**
   - 기본 → 조건 추가 → 여러 기능 조합
   - 같은 주제로 난이도를 높여갈 수 있음

### 난이도별 프롬프트 예시

| 난이도 | 프롬프트 예시 |
|-------|-------------|
| 초급 | "숫자 2개를 입력받아 더한 결과를 출력하는 코드" |
| 중급 | "숫자 2개를 입력받아 사칙연산 중 선택해서 계산하는 코드" |
| 고급 | "계산기 프로그램을 만들어줘. 연속 계산, 기록 저장 기능 포함" |

## 4. 목표별 구성 비율

각 목표는 약 15~20분 분량으로 구성합니다.

| 단계 | 시간 | 비율 |
|------|------|------|
| 오늘 배울 것 | 1~2분 | 10% |
| AI에게 요청하기 | 2~3분 | 15% |
| 코드 이해하기 | 5~7분 | 35% |
| 수정해보기 | 5~7분 | 35% |
| 정리 | 1분 | 5% |

**핵심: 실습(70%)이 이론(30%)보다 많아야 함**

## 5. 난이도별 교안 특징

### 초급 (15일)

| 항목 | 내용 |
|------|------|
| **대상** | 프로그래밍 처음 입문자 |
| **프롬프트 특징** | 한 가지 개념만 사용, 결과가 바로 보임 |
| **코드 길이** | 5~15줄 |
| **수정 과제** | 숫자/문자만 바꾸기, 출력 형식 변경 |
| **핵심 키워드** | print, input, if, for, 리스트, 함수, 파일 |

### 중급 (30일)

| 항목 | 내용 |
|------|------|
| **대상** | 기본 문법을 아는 학습자 |
| **프롬프트 특징** | 여러 개념 조합, 실용적인 프로그램 |
| **코드 길이** | 20~50줄 |
| **수정 과제** | 기능 추가, 에러 처리 추가, 구조 개선 |
| **핵심 키워드** | 클래스, 예외처리, 모듈, API, 웹스크래핑 |

### 고급 (45일)

| 항목 | 내용 |
|------|------|
| **대상** | 실전 프로젝트를 원하는 학습자 |
| **프롬프트 특징** | 프로젝트 단위, 설계부터 구현까지 |
| **코드 길이** | 50줄 이상, 여러 파일 |
| **수정 과제** | 리팩토링, 성능 개선, 기능 확장 |
| **핵심 키워드** | 디자인패턴, 동시성, DB, Flask, 데이터분석 |

## 8. 예시: Day 4 (조건문) 교안

```markdown
### 목표 1: if문으로 조건 분기하기

#### 오늘 배울 것
Python에서 조건에 따라 다른 코드를 실행하려면 **if문**을 사용합니다.

#### AI에게 요청하기
"숫자를 입력받아서 60점 이상이면 '합격', 미만이면 '불합격'을 출력하는 Python 코드 만들어줘"

#### 코드 이해하기
AI가 만든 코드를 보면서 확인하세요:
- "if 다음에 조건(score >= 60)을 쓰는구나"
- "조건 끝에 콜론(:)을 붙이는구나"
- "들여쓰기(4칸)로 실행할 코드를 구분하는구나"
- "else:는 조건이 거짓일 때 실행되는구나"

#### 수정해보기
1. 기준 점수를 70점으로 바꿔보세요
2. '합격'을 '축하합니다! 합격입니다'로 바꿔보세요

#### 정리
- 조건 분기: if 조건: / else:
- 필수: 콜론(:)과 들여쓰기(4칸)
- 다음에 쓸 키워드: "if문", "조건 분기"
```
