# VSCode에서 C 프로그램 작성 및 실행 동영상 시나리오

**영상 제목:** VSCode에서 C언어 코딩하기 | 컴파일부터 실행까지 완벽 가이드
**예상 길이:** 약 12~15분
**대상:** C언어 입문자, VSCode로 C 개발을 시작하려는 분
**사전 조건:** VSCode 설치 완료

**시간:** 약 45초

### 나레이션 대본

```
안녕하세요!

오늘은 VS Code에서 C언어 프로그램을

C언어는 운영체제, 임베디드 시스템, 게임 엔진 등

이 영상에서는 컴파일러 설치부터

VS Code가 이미 설치되어 있다면
```

## 본문 (화면녹화 + AI 나레이션)

### Scene 1: C/C++ 확장 설치
**시간:** 약 1분 30초

#### 화면 동작
1. VS Code 실행
2. 왼쪽 사이드바에서 확장(Extensions) 아이콘 클릭 (Ctrl+Shift+X)
3. 검색창에 "C/C++" 입력
4. Microsoft의 "C/C++" 확장 선택
5. "Install" 버튼 클릭
6. 설치 완료 확인

#### 나레이션
```
먼저 VS Code에서 C언어를 사용하기 위한 확장을 설치합니다.

VS Code를 실행하고, 왼쪽에 있는 블록 모양 아이콘을 클릭하세요.
키보드 단축키 Ctrl + Shift + X를 눌러도 됩니다.

검색창에 C/C++을 입력합니다.

첫 번째로 나오는 Microsoft에서 만든 C/C++ 확장을 선택하고
Install 버튼을 클릭하세요.

이 확장은 코드 자동완성, 문법 강조, 디버깅 등
C언어 개발에 필요한 기능을 제공합니다.

설치가 완료될 때까지 잠시 기다려주세요.
```

### Scene 2: MinGW 컴파일러 다운로드
**시간:** 약 1분 30초

#### 화면 동작
1. 웹 브라우저 실행
2. 주소창에 `sourceforge.net/projects/mingw` 입력
   (또는 `winlibs.com` - 더 최신 버전)
3. 다운로드 페이지 확인
4. "Download" 버튼 클릭

#### 나레이션
```
VS Code는 코드 편집기일 뿐,
C언어를 컴파일하려면 별도의 컴파일러가 필요합니다.

Windows에서는 MinGW라는 컴파일러를 많이 사용합니다.

웹 브라우저를 열고 sourceforge.net/projects/mingw를 입력하세요.

또는 winlibs.com에서 더 최신 버전을 다운로드할 수도 있습니다.

다운로드 페이지에서 Download 버튼을 클릭하세요.

파일 크기가 크기 때문에 다운로드에 시간이 걸릴 수 있습니다.
```

---

### Scene 3: MinGW 설치
**시간:** 약 2분

#### 화면 동작
1. 다운로드된 설치 파일 실행
2. 설치 경로 선택 (예: C:\MinGW)
3. 설치할 패키지 선택 (mingw32-gcc-g++ 필수)
4. Installation > Apply Changes
5. 설치 완료 확인

#### 나레이션
```
다운로드가 완료되면 설치 파일을 실행합니다.

설치 경로는 기본값을 사용하거나
C드라이브 바로 아래에 MinGW 폴더를 만들어 설치하는 것이 좋습니다.
경로가 간단해야 나중에 환경변수 설정이 편합니다.

설치 관리자가 열리면 필요한 패키지를 선택해야 합니다.
mingw32-base와 mingw32-gcc-g++를 선택하세요.
이 두 가지가 C와 C++ 컴파일에 필요한 핵심 패키지입니다.

선택했으면 Installation 메뉴에서 Apply Changes를 클릭합니다.

패키지 다운로드와 설치가 진행됩니다.
완료될 때까지 잠시 기다려주세요.
```

---

### Scene 4: 환경변수 설정 (PATH 추가)
**시간:** 약 2분

#### 화면 동작
1. Windows 검색에서 "환경 변수" 검색
2. "시스템 환경 변수 편집" 클릭
3. "환경 변수" 버튼 클릭
4. 시스템 변수에서 "Path" 선택 후 "편집"
5. "새로 만들기" 클릭
6. MinGW bin 경로 추가 (예: C:\MinGW\bin)
7. 확인 버튼으로 모든 창 닫기

#### 나레이션
```
이제 가장 중요한 단계입니다.
컴파일러를 어디서든 실행할 수 있도록 환경변수를 설정해야 합니다.

Windows 검색창에 "환경 변수"를 입력하고
"시스템 환경 변수 편집"을 클릭하세요.

시스템 속성 창이 열리면 아래쪽의 "환경 변수" 버튼을 클릭합니다.

시스템 변수 목록에서 Path를 찾아 선택하고 "편집"을 클릭하세요.

"새로 만들기" 버튼을 클릭하고
MinGW를 설치한 경로에 bin 폴더를 추가합니다.
예를 들어 C:\MinGW\bin 이렇게 입력합니다.

확인 버튼을 눌러 모든 창을 닫습니다.

이 설정을 완료해야 VS Code 터미널에서 gcc 명령어를 사용할 수 있습니다.
```

### Scene 8: Hello World 프로그램 작성
**시간:** 약 2분

#### 화면 동작
1. 코드 입력:
```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
```
2. 코드 자동완성 기능 보여주기
3. Ctrl+S로 저장

#### 나레이션
```
가장 기본적인 Hello World 프로그램을 작성해보겠습니다.

먼저 #include <stdio.h>를 입력합니다.
이것은 표준 입출력 라이브러리를 포함시키는 코드입니다.
printf 함수를 사용하려면 반드시 필요합니다.

다음 줄에 int main() 괄호 열고 닫고 중괄호를 입력합니다.
main 함수는 프로그램의 시작점입니다.

중괄호 안에 printf 괄호 열고 큰따옴표
Hello, World! 역슬래시 n 큰따옴표 닫고 괄호 닫고 세미콜론을 입력합니다.
역슬래시 n은 줄바꿈을 의미합니다.

마지막으로 return 0 세미콜론을 입력합니다.
이것은 프로그램이 정상적으로 종료되었음을 알려줍니다.

Ctrl + S를 눌러 파일을 저장합니다.
```

### Scene 10: 컴파일하기
**시간:** 약 1분 30초

#### 화면 동작
1. 터미널에 `gcc hello.c -o hello` 입력
2. 엔터 후 결과 확인
3. 탐색기에서 hello.exe 파일 생성 확인

#### 나레이션
```
이제 C 프로그램을 컴파일해보겠습니다.

터미널에 gcc hello.c -o hello라고 입력하고 엔터를 누르세요.

이 명령어의 의미를 설명드리면,
gcc는 컴파일러 이름입니다.
hello.c는 컴파일할 소스 파일입니다.
-o hello는 출력 파일 이름을 hello로 지정하는 옵션입니다.

엔터를 누르면 컴파일이 진행됩니다.
아무 메시지가 없으면 컴파일이 성공한 것입니다.

왼쪽 탐색기를 보시면 hello.exe 파일이 새로 생성된 것을 확인할 수 있습니다.
이것이 실행 가능한 프로그램 파일입니다.
```

```
PowerShell을 사용하는 경우 ./hello로 입력해도 됩니다.

엔터를 누르면 화면에 Hello, World!가 출력됩니다.

축하합니다!
첫 번째 C 프로그램을 성공적으로 작성하고 실행했습니다.
```

### Scene 12: 코드 수정 및 재컴파일
**시간:** 약 1분 30초

#### 화면 동작
1. 코드 수정 (메시지 변경 또는 여러 줄 출력)
```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    printf("C언어를 시작합니다!\n");
    printf("VS Code로 코딩하기\n");
    return 0;
}
```
2. Ctrl+S로 저장
3. 다시 컴파일: `gcc hello.c -o hello`
4. 실행: `.\hello`
5. 변경된 결과 확인

#### 나레이션
```
코드를 수정하고 다시 실행해보겠습니다.

printf 문을 몇 줄 더 추가해볼게요.
"C언어를 시작합니다!"와 "VS Code로 코딩하기"를 출력하도록 하겠습니다.

코드를 수정했으면 Ctrl + S로 저장합니다.

중요한 점은, 코드를 수정하면 반드시 다시 컴파일해야 합니다.
터미널에서 위쪽 화살표를 누르면 이전 명령어가 나옵니다.
gcc hello.c -o hello를 다시 실행하세요.

컴파일이 완료되면 .\hello로 다시 실행합니다.

이번에는 세 줄이 출력되는 것을 확인할 수 있습니다.
```

### Scene 13: 컴파일 오류 해결하기
**시간:** 약 1분 30초

#### 화면 동작
1. 일부러 오류 발생시키기 (세미콜론 빠뜨리기)
2. 컴파일 시도
3. 오류 메시지 확인
4. 오류 수정
5. 다시 컴파일 및 성공

#### 나레이션
```
프로그래밍을 하다 보면 오류가 발생할 수 있습니다.
오류를 해결하는 방법도 알아보겠습니다.

일부러 세미콜론을 하나 지워보겠습니다.
이 상태로 저장하고 컴파일해봅니다.

터미널에 오류 메시지가 표시됩니다.
expected semicolon, 즉 세미콜론이 필요하다는 메시지입니다.

오류 메시지를 잘 읽어보면 몇 번째 줄에서 오류가 발생했는지 알 수 있습니다.

해당 줄로 가서 세미콜론을 다시 추가합니다.
저장하고 다시 컴파일하면 오류 없이 성공합니다.

처음에는 오류가 많이 발생하지만 걱정하지 마세요.
오류 메시지를 읽고 수정하는 것도 프로그래밍의 일부입니다.
```

### Scene 14: 유용한 단축키 소개
**시간:** 약 1분

#### 화면 동작
1. 각 단축키 시연
   - Ctrl+S: 저장
   - Ctrl+`: 터미널 열기/닫기
   - Ctrl+Shift+B: 빌드 작업 실행 (설정 시)
   - F5: 디버깅 시작

#### 나레이션
```
VS Code에서 C언어 개발할 때 유용한 단축키를 알려드리겠습니다.

Ctrl + S는 파일 저장입니다.
코드를 수정할 때마다 자주 저장하는 습관을 들이세요.

Ctrl + 백틱은 터미널을 열고 닫는 단축키입니다.

Ctrl + Shift + B는 빌드 작업을 실행합니다.
나중에 tasks.json을 설정하면 이 단축키로 바로 컴파일할 수 있습니다.

F5는 디버깅을 시작하는 단축키입니다.
디버깅 설정은 별도의 영상에서 다루겠습니다.
```
